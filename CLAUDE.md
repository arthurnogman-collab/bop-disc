# Project Rules

## Workflow Rules

1. **Always commit and push after making changes.** Don't wait to be asked — commit and push to deploy automatically via Render.

2. **Always sync files to the preview location after edits.** The preview server runs from `C:\Users\arthu\bop-disc\`, not `C:\Users\arthu\bop\bop-disc\`. Copy changed files there after every edit so the preview reflects the latest code.

3. **Always test features after implementing.** Verify changes work in the preview (reload, inspect, check console for errors) before telling the user it's done. Never claim something is fixed without confirming it.

4. **Always ask questions if something is ambiguous.** Don't guess or invent solutions when the intent is unclear. Ask the user to clarify before proceeding.

5. **Always use the highest effort level.** Use the most capable models and thorough approaches. Implementing things wrong causes more trouble than not implementing at all. Take the time to get it right the first time.
